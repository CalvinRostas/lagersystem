# **Requirement 10: Generate QR Code**

**Title:** Generate QR Code during Item / Storage Location Creation
**Number:** SRS-010
**State:** approved
**Priority:** high

# Header

**When** a user creates a new item / storage location,
**the system shall** automatically generate a QR code containing the unique identifier of the item / storage location.

# Acceptance Criteria:

- **Pre-condition:**
  - The user has initiated the item / storage location creation process.
  - All mandatory item / storage location attributes have been provided and validated.

- **Content:**
  - A unique item / storage location ID is generated by the system during the creation process.
  - The QR code encodes this unique item / storage location ID.
  - The QR code is displayed immediately after successful item / storage location creation.

- **Usability:**
  - The QR code is displayed in sufficient size and resolution for reliable scanning.
  - The user can download or save the QR code.

- **Consistency:**
  - The encoded data in the QR code matches exactly the newly generated item / storage location ID.
  - The stored item / storage location ID in the local storage is identical to the encoded QR content.

- **Error Handling:**
  - If mandatory input data is missing or invalid, the QR code is not generated.
  - If item / storage location creation fails, no QR code is generated and an appropriate error message is displayed.

- **Verifiability:**
  - After creation, scanning the QR code returns exactly the stored item / storage location.
  - The generated QR code can be validated using an external QR code scanning application.
